# Apliacion 01

En esta práctica implementamos un sistema de gestión estudiantil utilizando una Tabla Hash con direccionamiento abierto, donde el reto principal fue resolver colisiones buscando huecos alternativos en el mismo arreglo (sondeo) en lugar de usar listas externas.  Aprendí que la eficiencia del sistema, idealmente $O(1)$, depende críticamente de mantener un factor de carga bajo para evitar degradar el rendimiento a una búsqueda secuencial. Sin embargo, el desafío técnico más significativo fue la eliminación de registros, donde comprendí que no se pueden vaciar las celdas simplemente, sino que es necesario implementar un "borrado lógico" (marcas o tombstones) para no romper la cadena de búsqueda de otros elementos que colisionaron previamente.
