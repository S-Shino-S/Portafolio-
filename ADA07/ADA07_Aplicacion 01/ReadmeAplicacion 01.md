# Apliacion 01
Registro de Estudiantes con Colisiones Controladas (Hash Table)

¿En qué consistió la actividad?
El objetivo principal de esta práctica fue implementar una estructura de datos de acceso directo: la Tabla Hash. La actividad consistió en desarrollar un sistema en C/C++ para gestionar registros de estudiantes, utilizando su matrícula (un número de hasta 6 dígitos) como la clave única (key) para el almacenamiento.El reto técnico no fue solo la inserción de datos, sino la gestión eficiente de conflictos mediante Direccionamiento Abierto (Open Addressing). Tuve que programar la lógica para que, cuando la función hash dirigiera a dos estudiantes a la misma posición del arreglo, el sistema buscara la siguiente casilla disponible (usando sondeo lineal o cuadrático) en lugar de usar listas enlazadas externas. El sistema final permite agregar, buscar y eliminar alumnos manteniendo una complejidad de tiempo cercana a $O(1)$.
Reflexión Personal
¿Qué se aprendió?
Comprendí la importancia crítica de una buena Función Hash (generalmente usando el operador módulo %) para distribuir uniformemente las claves y minimizar colisiones. Aprendí cómo el Load Factor (factor de carga) afecta el rendimiento: si la tabla se llena demasiado, el direccionamiento abierto se vuelve lento, pareciéndose a una búsqueda secuencial. También entendí la diferencia entre el encadenamiento (usar listas) y el direccionamiento abierto (buscar huecos en el mismo arreglo), siendo este último más eficiente en uso de caché pero más complejo de gestionar cuando la tabla está llena.
Problemas y desafíos: El problema más complejo y técnico fue la función de Eliminar (Delete). En el direccionamiento abierto, no se puede simplemente borrar una celda (ponerla en NULL) porque eso rompería la cadena de búsqueda para otros elementos que colisionaron y fueron movidos más adelante. Tuve que implementar una estrategia de "borrado lógico" (usando una bandera o estado DELETED / TOMBSTONE), lo cual complicó la lógica de búsqueda e inserción posterior. Otro desafío fue evitar bucles infinitos durante el sondeo cuando la tabla estaba casi llena.
